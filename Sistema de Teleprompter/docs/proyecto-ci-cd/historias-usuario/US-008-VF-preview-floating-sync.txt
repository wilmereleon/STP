# US-008: Sincronización Preview ↔ Ventana Flotante | Epic: Vista Flotante | 🔴 CRÍTICA | 5 pts | Sprint 1

## 📋 Descripción

**Como** operador del teleprompter  
**Quiero** que el Preview y la Ventana Flotante estén perfectamente sincronizados  
**Para** que ambas muestren exactamente el mismo contenido y posición de scroll en todo momento

**Origen:** Bug VF-3 del Reporte de Pruebas (23 Oct 2025)  
**Problema:** Preview y FloatingWindow se dessincronizan al hacer scroll o cambiar contenido

---

- [ ] Scroll sincronizado bidireccionalmente (Preview → Floating y viceversa)
- [ ] Mismo contenido mostrado en ambas vistas simultáneamente

- [ ] Play/Pause sincronizados entre ventanas
- [ ] Velocidad de scroll idéntica en ambas

- [ ] Latencia máxima de sincronización: 16ms (1 frame @ 60fps)

- [ ] Jump markers respetados en ambas ventanas

## 📝 Notas

### Arquitectura del ScrollSyncService

```typescript
class ScrollSyncService {
  private views: Map<string, HTMLElement> = new Map();
  private isSyncing = false; // Evitar loops infinitos
  
  registerView(viewId: string, element: HTMLElement) {
    this.views.set(viewId, element);
  }
  
  unregisterView(viewId: string) {
    this.views.delete(viewId);
  }
  
  syncScroll(sourceViewId: string, scrollTop: number) {
    if (this.isSyncing) return; // Evitar loop
    
    this.isSyncing = true;
    
    this.views.forEach((element, viewId) => {
      if (viewId !== sourceViewId) {
        element.scrollTop = scrollTop;
      }
    });
    
    this.isSyncing = false;
  }
}

export const scrollSyncService = new ScrollSyncService();
```

### Throttle para Performance

```typescript
const throttledSync = throttle((viewId, scrollTop) => {
  scrollSyncService.syncScroll(viewId, scrollTop);
}, 16); // 60fps = 16ms por frame
```

### Consideraciones

1. **Loop Infinito:** Usar flag `isSyncing` para evitar que el sync cause otro sync
2. **Performance:** Throttle a 60fps (16ms) para evitar sobrecarga
3. **Precisión:** scrollTop debe ser exacto (no aproximado)
4. **Edge Cases:** Manejar cuando una vista se cierra/abre
5. **RequestAnimationFrame:** Considerar usar RAF para scroll suave

### Métricas de Éxito

- ✅ Latencia < 16ms (60fps)
- ✅ 0 desincronizaciones reportadas en QA
- ✅ CPU usage < 10% durante scroll
- ✅ Sin jank visual (frames dropped)
